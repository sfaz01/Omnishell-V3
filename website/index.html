Safety First

Five layers
between you
and rm -rf /.

AI-assisted shells are powerful. OmniShell treats that power with the respect it deserves — a defense-in-depth architecture that catches mistakes at every stage.

● LLM
● Regex
● Sudo
● Confirm
01

LLM-level instruction

The system prompt explicitly tells the model to return SAFE_MODE_ERROR for destructive requests. The AI refuses before you ever see a command.

02

Regex blocklist

Hard-coded patterns block rm -rf /, fork bombs, disk overwrites — regardless of what the LLM outputs. The model cannot bypass this.

03

Sudo warnings

Commands using sudo or destructive flags (-f, --force, dd) trigger explicit visual warnings.

04

Manual confirmation

Every command requires explicit y/N approval before execution. The default is always N.

05

Dry-run mode

Pass --dry-run to preview what would execute without any side effects. Perfect for unfamiliar commands.

Execution lifecycle

From sentence to stdout.

Every command travels the same eight-step path. Predictable. Debuggable. Safe.

STEP 01

You type naturally

Plain English (or any language the LLM understands). No flags, no syntax, no man pages.

$ omni "list running containers"
STEP 02

Context assembled

OS, shell, recent memory, and your prompt are merged into a single LLM request.

# memory.py + prompts.py
context = {
"os": "linux", "shell": "zsh",
"history": last_n(3), "prompt": user_input
}
STEP 03

LLM generates command

The provider (Groq by default) returns a structured response with command + explanation.

{
"command": "docker ps",
"explanation": "Lists all running Docker containers",
"safe": true
}
STEP 04

Regex blocklist check

The generated command is matched against hard-coded dangerous patterns. This runs locally and cannot be bypassed by the LLM.

# safety.py
if matches_blocklist(command):
return SAFE_MODE_ERROR(command)
STEP 05

Destructive flag detection

Sudo, -rf, --force, dd, and friends trigger prominent visual warnings in the Rich UI.

STEP 06

User confirmation

The command and explanation are displayed. You press y to run, n to abort, or e to edit.

→ docker ps
Lists all running Docker containers

Run this command? [y/N/e]:
STEP 07

Execution + streaming

Subprocess runs the command. Output streams live through Rich for syntax highlighting and progress.

STEP 08

History + memory persisted

Prompt, generated command, exit code, and output are stored in SQLite. Conversational memory updates for follow-ups.

# history.py + memory.py
save_to_sqlite(prompt, command, exit_code, output)
update_memory(prompt, command, output)
